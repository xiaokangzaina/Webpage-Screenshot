# 网页实时获取截图

AstrBot 网页截图插件。支持定时截取网页并推送到群聊，也支持在目标会话内发送“状态”立即获取截图。

## v1.1.1

修复：

- 前端配置不再写入 AstrBot 普通配置页的 `tasks` 模板列表。
- 前端配置改为写入插件独立数据文件：`data/webpage_screenshot_tasks.json`。
- 避免普通配置页 `template_list` 因模板标识兼容问题提示“找不到对应模板”。
- 插件运行时会自动读取独立数据文件中的群截图任务。

## v1.1.0 重大更新

- 新增 Web 前端配置页。
- 左侧显示 QQ 群列表，进入页面自动同步。
- 最近配置过的群聊自动置顶。
- 点击群聊后配置该群的截图任务。
- 原全局配置已融入单群配置，每个群都可以独立设置。
- 米黄色主题 UI。
- 群列表超出高度时自动滚动。

## 单群配置项

| 配置项 | 说明 |
|---|---|
| 启用该群截图任务 | 是否启用该群定时截图。 |
| 任务名称 | 用于日志和消息模板。 |
| 网页地址 | 要截图的完整 URL。 |
| 推送间隔分钟 | 定时截图间隔。 |
| 发送截图文字 | 是否发送截图附带文字。 |
| 截图文字 | 支持 `{name}` 和 `{url}`。 |
| 引用和艾特发送人 | 手动“状态”截图时引用并艾特。 |
| 失败通知 | 定时截图失败时通知该群。 |
| 浏览器宽度 | 截图浏览器宽度。 |
| 浏览器高度 | 截图浏览器高度。 |
| 页面等待秒数 | 页面加载后等待时间。 |
| 页面超时秒数 | 页面最大加载等待。 |

## 使用

1. 打开 AstrBot Dashboard。
2. 进入插件管理。
3. 打开“网页实时获取截图”的配置页面。
4. 左侧选择群聊。
5. 填写网页地址并启用该群任务。
6. 保存配置。

在已配置的目标会话内发送：

```text
状态
```

可立即获取一次截图。

## 依赖

```bash
pip install -r requirements.txt
playwright install chromium
```

## 注意

- `target_id` 默认留空，不包含真实群号或 QQ 号。
- API Key、Token、Cookie 不需要配置。
- 本地配置文件可能包含你的真实群号，发布包不会包含这些本地配置。
