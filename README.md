# 网页实时获取截图

按固定间隔打开指定网页，截取页面并发送到指定群聊或私聊。

![插件头像](logo.png)

## 功能

- 定时截取网页
- 支持多个截图任务
- 支持群聊和私聊推送
- 支持自定义浏览器窗口宽高
- 支持页面加载等待时间
- 支持自定义截图说明文字
- 支持在指定会话内手动获取截图

## 使用场景

适合监控状态页、面板页、公告页、数据看板、服务器页面等需要定时查看的网页。

## 安装依赖

插件依赖 Playwright。

在 AstrBot 的 Python 环境中执行：

```bash
pip install -r requirements.txt
playwright install chromium
```

## 配置说明

在插件配置中添加截图任务。

每个任务包含：

| 配置项 | 说明 |
| --- | --- |
| 任务名称 | 发送文字中使用的名称 |
| 启用此任务 | 是否运行该任务 |
| 网页地址 | 要截图的完整网页地址 |
| 发送到哪里 | 群聊或私聊 |
| 群号或 QQ 号 | 群聊填群号，私聊填 QQ 号 |
| 截图附带文本 | 每个任务单独的发送文字 |
| 每隔多少分钟发送一次 | 定时推送间隔 |
| 浏览器窗口宽度 | 截图窗口宽度 |
| 浏览器窗口高度 | 截图窗口高度 |
| 页面打开后等待秒数 | 页面加载慢时可以调大 |
| 页面加载超时秒数 | 打开网页的最长等待时间 |

## 自定义发送文本

全局和单个任务都可以设置截图附带文本。

支持变量：

```text
{name} 任务名称
{url} 网页地址
```

示例：

```text
图片已发送，目前状态
```

或：

```text
{name} 当前状态截图
```

发送时文字在上方，图片在下方。

## 手动获取截图

只在任务配置的目标群聊或私聊内生效。

需要使用唤醒词或艾特机器人。

示例：

```text
/状态
```

或：

```text
@机器人 状态
```

其他群聊不会触发截图。

## 注意事项

- 网页需要机器人所在机器可以正常访问。
- 如果网页加载较慢，调大等待秒数。
- 如果推送失败，检查机器人是否能向目标群聊或用户发送消息。
- aiocqhttp 场景下，群聊目标填写群号，私聊目标填写 QQ 号。
